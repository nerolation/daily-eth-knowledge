Lesson 28

Committee Index Aggregation (EIP-7549)

March 3, 2026

📍 Spec Location

  • electra/beacon-chain.md → Modified Attestation, IndexedAttestation, get_attesting_indices
  • EIP-7549: "Move committee index outside Attestation"

💡 Core Concept

EIP-7549 fundamentally changes how attestations aggregate. The key insight: every slot, all committees vote on the same thing — same LMD GHOST vote (block root, slot) and same FFG vote (source, target). The only difference? Committee index.

The Problem (Pre-Electra)

# Phase 0 AttestationData - committee index INSIDE the signed data
class AttestationData(Container):
    slot: Slot
    index: CommitteeIndex      # ← This is signed!
    beacon_block_root: Root
    source: Checkpoint
    target: Checkpoint

Because index is part of the signed AttestationData, attestations from different committees produce different signing roots — even when they're voting for identical (slot, block, source, target).

Result: You can only aggregate attestations from the same committee. With 64 committees per slot, you need to verify ~1,366 attestations minimum to reach 2/3 threshold.

🛠 The Electra Solution

Move index outside the signed message. Now attestations with the same vote can aggregate across ALL committees:

# Electra Attestation - committee info OUTSIDE signed data
class Attestation(Container):
    # Spans ALL committees now
    aggregation_bits: Bitlist[MAX_VALIDATORS_PER_COMMITTEE * MAX_COMMITTEES_PER_SLOT]
    data: AttestationData    # index field ignored (set to 0)
    signature: BLSSignature
    # NEW: Which committees are included
    committee_bits: Bitvector[MAX_COMMITTEES_PER_SLOT]

The committee_bits field is a 64-bit bitvector indicating which committees' validators are represented in this attestation. The aggregation_bits now spans across all included committees sequentially.

Helper to decode which committees

def get_committee_indices(committee_bits: Bitvector) -> Sequence[CommitteeIndex]:
    return [CommitteeIndex(index) for index, bit in enumerate(committee_bits) if bit]

Modified get_attesting_indices

def get_attesting_indices(state: BeaconState, attestation: Attestation) -> Set[ValidatorIndex]:
    output: Set[ValidatorIndex] = set()
    committee_indices = get_committee_indices(attestation.committee_bits)
    committee_offset = 0
    for committee_index in committee_indices:
        committee = get_beacon_committee(state, attestation.data.slot, committee_index)
        committee_attesters = set(
            attester_index
            for i, attester_index in enumerate(committee)
            if attestation.aggregation_bits[committee_offset + i]
        )
        output = output.union(committee_attesters)
        committee_offset += len(committee)
    return output

❓ Why This Way

  1. Verification efficiency: With cross-committee aggregation, minimum attestations to verify 2/3 drops from ~1,366 to ~22 (factor of 62!)
  2. ZK circuit optimization: Casper FFG proofs now need far fewer pairing operations
  3. Block space efficiency: Changed limits reflect new reality:
    • MAX_ATTESTATIONS_ELECTRA = 8 (down from 128)
    • MAX_ATTESTER_SLASHINGS_ELECTRA = 1 (down from 2)
    • But each attestation can carry ~64x more votes!
  4. Network resilience: Chain with only 1/8 online proposers can still include all votes. Pre-Electra: needed 1/2 online proposers.

Size Analysis

🔗 Connections

  • SingleAttestation: New container for individual validator attestations over gossip
  • IndexedAttestation: Now supports MAX_VALIDATORS_PER_COMMITTEE * MAX_COMMITTEES_PER_SLOT indices
  • Slashing detection: AttesterSlashing still works, just with bigger potential index lists

⚠️ Gotchas

  1. Fork transition gap: First Electra block can't include pre-fork attestations (container mismatch). One slot of FFG votes lost — acceptable tradeoff vs. dual-format complexity.
  2. Gossip mutation risk: Since committee_bits isn't signed, attackers could mutate it on p2p. Mitigated by: "attestation.attester_index must be in get_beacon_committee(state, slot, index)" — mutation fails this check.
  3. Bitvector vs Bitlist: committee_bits uses fixed-size Bitvector[64] — half the size of equivalent Bitlist. State transition validates that set bits match actual committee count.
  4. The index field lives on: AttestationData.index isn't removed — it's set to zero. Cleaner than Optional types from EIP-7495, and AttesterSlashing logic stays simpler.

🎯 Key Takeaway

EIP-7549 is architectural cleanup with massive efficiency gains. Same votes, same security, but aggregate them together instead of keeping them artificially separated by committee assignment.

The invariant: If validators vote for the same thing, their signatures should combine. Pre-Electra violated this by signing over committee index. Now it doesn't.
