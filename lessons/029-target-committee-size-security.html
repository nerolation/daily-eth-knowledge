📚 Lesson 29: Target Committee Size and Security

Spec: phase0/beacon-chain.md → Preset → Misc

💡 Core Concept

Why is TARGET_COMMITTEE_SIZE = 128? This isn't arbitrary—it's derived from probability theory to ensure committee-based consensus remains secure even with an adversary controlling up to 1/3 of stake.

The threat model: if an adversary controls 1/3 of all validators globally, what's the probability they control >1/2 of a random committee? If committees are too small, random sampling variance can give the adversary majority by luck.

The spec note states:

For the safety of committees, TARGET_COMMITTEE_SIZE exceeds the recommended minimum committee size of 111

That 111 comes from Vitalik's 2018 sharding analysis using the birthday bound—a probabilistic guarantee that failure probability stays below 2-40 (roughly one-in-a-trillion).

🧮 The Math: Birthday Bound Analysis

Let's derive why ~111+ validators provides 2-40 failure probability.

Setup:

Total validators (N):Large (e.g., 500k+) Adversary fraction (f):1/3 Committee size (n):Variable "Failure":Adversary gets >n/2 seats

This is a hypergeometric distribution, but for large N, approximates binomial:

P(adversary ≥ n/2 in committee) ≈ P(Binomial(n, 1/3) ≥ n/2)

Using Chernoff bound for deviation from mean (n/3):

P(X ≥ n/2) = P(X ≥ (1+δ)μ) where μ = n/3, δ = 0.5
≤ exp(-μδ²/3) = exp(-n/36)

For 2-40 ≈ 10-12 failure probability, the tighter Chernoff bound gives n ≈ 111. The spec rounds up to 128 (a nice power of 2) for extra margin.

🛠 How the Spec Enforces This

# Preset constants
TARGET_COMMITTEE_SIZE = uint64(2**7)  # = 128
MAX_COMMITTEES_PER_SLOT = uint64(2**6)  # = 64
SLOTS_PER_EPOCH = uint64(2**5)  # = 32

The get_committee_count_per_slot function dynamically adjusts:

def get_committee_count_per_slot(state, epoch):
    return max(
        uint64(1),
        min(
            MAX_COMMITTEES_PER_SLOT,
            len(active_validators) // SLOTS_PER_EPOCH 
                                   // TARGET_COMMITTEE_SIZE,
        ),
    )

Key Logic:

  1. Divide active validators across 32 slots
  2. Divide again by 128 to get committee count
  3. Cap at 64 committees max, floor at 1 committee min

Example with 500k validators:

committees_per_slot = min(64, 500000 / 32 / 128)
                   = min(64, 122)
                   = 64 committees per slot
committee_size ≈ 500000 / (32 × 64) ≈ 244 validators

Result: committees are ~244 validators—well above the 128 target.

❓ Why This Design?

Why not just use ALL validators per slot?

  • BLS aggregation scales O(n) for verification
  • 500k validators × 32 slots = network can't handle it
  • Committees provide statistical security with bounded work

Why 2/3 honest majority matters for committees:

  • Attestations need to reach finality threshold
  • A corrupted committee can't finalize bad blocks alone
  • But it CAN delay or create fork choice confusion

The real security property:

For any given slot, with 1/3 adversary stake, the probability that ANY committee is adversary-majority is:

P(any bad) ≤ 64 × 2-40 ≈ 2-34

Still astronomically unlikely, but not zero. This is why Ethereum uses aggregated attestations across many committees, not single-committee decisions.

⚠️ Gotchas

1. Low validator counts: With <4096 validators (128 × 32), some committees WILL be smaller than 128. The protocol still works but with degraded statistical guarantees.
2. Assumes uniform random selection: The shuffling algorithm (swap-or-not with 90 rounds) must be unbiasable. RANDAO's 1-bit bias is considered acceptable given the margin.
3. 128 is TARGET, not guarantee: The spec ensures committees are at least this size when there are enough validators. MAX_VALIDATORS_PER_COMMITTEE = 2048 sets the upper bound.

Key Takeaway

TARGET_COMMITTEE_SIZE = 128 is a cryptoeconomic security parameter, not a performance tuning knob. It guarantees that even with 1/3 adversary stake, random sampling into committees almost never gives adversaries majority control. The 2-40 failure probability means you'd need to run the chain for longer than the universe's age to expect one unlucky committee.
